# 产品需求文档 (PRD): IconTint (图标变色大师)

**版本:** 1.0  
**状态:** 初稿  
**目标平台:** Web (Mobile Friendly)  
**部署环境:** Cloudflare Pages

---

## 1. 项目背景与目标
### 1.1 背景
开发者（尤其是小程序开发者）在处理 UI 状态（如 TabBar 激活态）时，频繁需要将单色图标转换为特定主题色。目前市面工具要么功能冗余、要么 UI 过时、要么带有广告。

### 1.2 目标
打造一个**极致简洁、现代化、无广告**的在线图标变色工具，支持 PNG 和 SVG，解决开发者“最后 1 公里”的切图痛点。

---

## 2. 用户角色与场景
* **开发者 A:** 正在写微信小程序，从 Iconfont 下了个黑色的 `home.png`，需要快速生成一个 `#07C160` 的 `home_active.png`。
* **设计师 B:** 需要快速给非矢量图标换色以适配不同的 PPT 主题。
* **独立开发者 C:** 希望有一个工具能批量处理一套图标的颜色，且不希望上传文件到服务器（保护隐私）。

---

## 4. 非功能需求 (Non-functional Requirements)

### 4.1 性能与部署
* **纯前端处理:** 所有图像处理在浏览器完成，不向服务器传输任何图片数据（**隐私保护 + 零服务器成本**）。
* **极速加载:** 首屏加载时间控制在 1s 内（Cloudflare CDN 优势）。
* **响应式:** 适配移动端，虽然主要用户是开发者，但要保证手机端也能紧急操作。

### 4.2 设计规范 (UI/UX)
* **风格:** 现代极简 (Modern Minimalist)，采用 Tailwind CSS。
* **交互:** 拖拽上传时有明显的边框高亮反馈。
* **视觉:** 玻璃拟态 (Glassmorphism) 效果，深色模式适配。

---

## 5. 核心技术逻辑 (逻辑参考)

### 5.1 PNG 变色逻辑 (Canvas)
1. 将图片绘制到隐藏的 Canvas。
2. 使用 `globalCompositeOperation = 'source-in'`。
3. 填充目标颜色。此操作会将非透明区域全部替换为新颜色，同时保留原始的抗锯齿透明度。

### 5.2 SVG 变色逻辑
1. 将 SVG 字符串解析为 DOM。
2. 遍历所有元素，若元素包含 `fill` 或 `stroke`（且非 `none`），则覆盖为目标色。
3. 重新序列化为数据 URL 进行预览和下载。

## 6. 页面结构规划 (Wireframe)

1.  **Header:** Logo (IconTint) + 语言切换 + GitHub 链接。
2.  **Hero Area:** 大标题“图标变色从未如此简单” + 简洁的一行功能描述。
3.  **Workspace (核心区):**
    * **左侧:** 上传区域（支持 Drag & Drop）。
    * **中间:** 颜色调节盘（Hex 输入 + 快速色板）。
    * **右侧:** 预览区域 + 下载按钮。
4.  **Batch Area (可选):** 底部显示已上传的图标列表。
5.  **Footer:** SEO 文本内容 + 友情链接。
